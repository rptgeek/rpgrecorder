# Requirements for TTRPG Session Recorder and Summarizer

## v1 Requirements (Minimum Viable Product)

These are the essential features for the initial release of the project.

### Authentication
- [x] **AUTH-01**: User can create, log in to, and manage their account securely.
  - **Outcome:** ✓ Validated - Implemented with Auth.js, credential provider, middleware route protection

### Session Management
- [x] **SESS-01**: User can create, edit, delete, and view their TTRPG sessions.
  - **Outcome:** ✓ Validated - Complete CRUD operations with server actions and API routes

### Audio Handling
- [x] **AUDIO-01**: System can record audio during a session.
  - **Outcome:** ✓ Validated - Browser-based recording with AudioRecorder component
- [x] **AUDIO-02**: User can upload audio files for processing.
  - **Outcome:** ✓ Validated - S3 presigned URLs for secure upload

### Transcription
- [x] **TRANS-01**: System automatically transcribes recorded/uploaded audio into text.
  - **Outcome:** ✓ Validated - AWS Transcribe integration with webhook completion
- [x] **TRANS-02**: System can identify and differentiate speakers in the transcription.
  - **Outcome:** ✓ Validated - AWS Transcribe speaker diarization enabled

### Review & Playback
- [x] **REVIEW-01**: User can play back session audio synchronized with its transcript.
  - **Outcome:** ✓ Validated - TranscriptDisplay component with timestamp-based synchronization

### GM Notes
- [x] **NOTES-01**: User can add basic manual notes to a session.
  - **Outcome:** ✓ Validated - NotesEditor component with rich text support

### Search
- [x] **SEARCH-01**: User can perform keyword searches within session transcripts.
  - **Outcome:** ✓ Validated - PostgreSQL full-text search with automatic indexing

### AI Summarization
- [x] **AI-01**: System automatically generates an AI-powered summary of the session.
  - **Outcome:** ✓ Validated - Claude 3.5 Sonnet with structured output (Executive Summary, NPCs, Items, Plot Hooks, Quotes)

### Dashboard & Campaign
- [x] **DASH-01**: System provides a dashboard with an overview of sessions within a campaign.
  - **Outcome:** ✓ Validated - Campaign management with dashboard and session listing

### Player Recaps
- [x] **RECAP-01**: System can generate player-specific recaps of a session.
  - **Outcome:** ✓ Validated - AI-powered recaps that filter GM-only information

### Session Metrics
- [x] **METRICS-01**: System can provide basic session metrics (e.g., session length).
  - **Outcome:** ✓ Validated - Duration, word count, speaker statistics with in-place renaming

## v2 Requirements (Deferred Features)

These features are planned for subsequent releases after the v1 product is established.

### Data Extraction & Tracking
- [ ] **DATA-01**: System can extract and track entities (e.g., characters, locations, items) across sessions.

### Search
- [ ] **SEARCH-02**: User can perform semantic searches within session data.

### Customization
- [ ] **CUSTOM-01**: User can customize parameters for summarization and recaps.

### Plot Identification
- [ ] **PLOT-01**: System can identify potential plot hooks and loose ends within sessions.

## Out of Scope

The following features are explicitly not planned for this project:

-   **Dice roll tracking and analysis:** Not a core focus for narrative review.
-   **Full Virtual Tabletop (VTT) functionality:** Focus remains on post-session analysis, not live gameplay platform.
-   **Over-reliance on manual tagging/categorization:** System aims for AI-driven automation.
-   **Real-time transcription:** Prioritizes accuracy over immediate transcription during live sessions.
-   **Complex social networking features:** Project is a utility for GMs, not a social platform.
